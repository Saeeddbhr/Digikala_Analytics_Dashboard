\# داشبورد تحلیل دیجی‌کالا



یک داشبورد جامع Power BI بر پایه داده‌های واقعی دیجی‌کالا، شامل تحلیل فروش، رفتار مشتری، کیفیت محصول و لجستیک.



\---



\## صفحات داشبورد



\### ۱. نمای کلی مدیریتی

شاخص‌های کلیدی عملکرد و روند درآمد برای تصمیم‌گیری مدیریتی.



\- \*\*GMV\*\* (ارزش ناخالص کالای فروخته‌شده) 

\- \*\*مشتری فعال\*\* — مشتریانی که در ۹۰ روز اخیر نسبت به آخرین تاریخ موجود در داده خرید داشته‌اند

\- \*\*میانگین ارزش سبد خرید (AOV)\*\*

\- نمودار روند درآمد با پیش‌بینی ۳ ماهه بر اساس DAX

\- ۱۰ محصول برتر بر اساس درآمد



\### ۲. هوش مشتری — RFM

سگمنت‌بندی مشتریان با استفاده از مدل RFM (تازگی، تکرار، ارزش مالی).



\- \*\*تازگی (Recency):\*\* تعداد روز از آخرین خرید نسبت به آخرین تاریخ در داده

\- \*\*تکرار (Frequency):\*\* تعداد سفارش‌های متمایز هر مشتری

\- \*\*ارزش مالی (Monetary):\*\* مجموع خرید هر مشتری

\- امتیازدهی ۱ تا ۵ به هر بعد و تخصیص سگمنت: Champions، Loyal Customers، Potential Loyalists، New Customers، At Risk، Can't Lose Them، Lost

\- نمودار پراکندگی: تکرار در برابر ارزش مالی با رنگ‌بندی بر اساس سگمنت

\- ارزش طول عمر مشتری (CLV) به تفکیک سگمنت

\- جدول `CustomerRFM` اختصاصی ساخته‌شده با `SUMMARIZE` در DAX برای محاسبه دقیق به ازای هر مشتری



\### ۳. محصول و موجودی

تحلیل ABC و دسته‌بندی عملکرد محصولات.



\- \*\*طبقه‌بندی ABC:\*\* رتبه‌بندی محصولات بر اساس سهم تجمعی درآمد (A = ۸۰٪ اول، B = ۱۵٪ بعدی، C = ۵٪ باقی‌مانده)

\- رتبه درآمدی و درصد تجمعی به عنوان Calculated Column روی جدول اختصاصی `ProductABC`

\- هیت‌مپ فروش: دسته‌بندی در برابر ماه با استفاده از Matrix Visual و قالب‌بندی شرطی پس‌زمینه

\- دسته‌بندی محصولات فعال در برابر راکد



\### ۴. کیفیت و رضایت مشتری

تحلیل رضایت مشتری بر اساس نظرات و امتیازات محصولات.



\- \*\*نرخ توصیه\*\* از داده‌های نظرات

\- \*\*نسبت رضایت:\*\* لایک‌ها / (لایک‌ها + دیسلایک‌ها)

\- ۱۰ محصول با بیشترین لایک و ۱۰ محصول با بیشترین دیسلایک

\- روند حجم نظرات به تفکیک دسته‌بندی

\- ساخته‌شده بر اساس داده `keifiat` متصل به `DimProduct`



\### ۵. لجستیک و جغرافیا

تحلیل مکانی توزیع سفارش‌ها در شهرهای ایران.



\- نقشه پُرشده و نقشه حبابی: حجم سفارش و GMV به تفکیک شهر

\- ۱۵ شهر برتر بر اساس تعداد سفارش و درآمد

\- تمرکز محصولات دسته A به تفکیک شهر (برای برنامه‌ریزی انبار)

\- هیت‌مپ الگوی سفارش: روز هفته در برابر ساعت



\### ۶. عملیات

شاخص‌های عملیاتی برای تیم‌های زنجیره تامین و تحویل.



\- روند و پیک حجم سفارش‌ها

\- تمرکز تقاضا به تفکیک شهر

\- تقاضای محصولات دسته A به تفکیک جغرافیا

\## مدل داده



مدل ستاره‌ای با یک جدول واقعیت و چهار جدول بعد.



```

&#x20;         DimDate

&#x20;            │

DimProduct ──┼── FactOrders ──── CustomerRFM

&#x20;            │

&#x20;         (دسته‌بندی از طریق DimProduct)

```



| جدول | نوع | توضیح |

|------|-----|-------|

| `FactOrders` | Fact | بیش از ۲ میلیون تراکنش سفارش |

| `DimProduct` | Dimension | کاتالوگ محصولات با دسته‌بندی |

| `DimDate` | Dimension | جدول تاریخ ساخته‌شده در DAX با `CALENDAR()` |

| `CustomerRFM` | Calculated | شاخص‌های RFM به ازای هر مشتری با `SUMMARIZE` |

| `ProductABC` | Calculated | رتبه‌بندی درآمدی محصولات با `SUMMARIZE` |

\## معیارهای DAX کلیدی



```dax

&#x20;مشتری فعال نسبت به آخرین تاریخ در داده

Active Customers =

CALCULATE(

&#x20;   DISTINCTCOUNT(FactOrders\[ID\_Customer]),

&#x20;   FILTER(

&#x20;       ALL(FactOrders),

&#x20;       FactOrders\[DateTime\_CartFinalize] >=

&#x20;           MAX(FactOrders\[DateTime\_CartFinalize]) - 90

&#x20;   )

)



\-- رشد GMV ماه به ماه

GMV vs Last Month =

VAR ThisMonth = \[GMV]

VAR LastMonth = CALCULATE(\[GMV], DATEADD(DimDate\[Date], -1, MONTH))

RETURN DIVIDE(ThisMonth - LastMonth, LastMonth)



\-- سگمنت‌بندی RFM (Calculated Column روی CustomerRFM)

RFM\_Segment =

SWITCH(TRUE(),

&#x20;   CustomerRFM\[R\_Score] >= 4 \&\& CustomerRFM\[F\_Score] >= 4, "Champions",

&#x20;   CustomerRFM\[R\_Score] >= 3 \&\& CustomerRFM\[F\_Score] >= 3, "Loyal Customers",

&#x20;   CustomerRFM\[R\_Score] >= 4 \&\& CustomerRFM\[F\_Score] <= 2, "New Customers",

&#x20;   CustomerRFM\[R\_Score] >= 3 \&\& CustomerRFM\[M\_Score] >= 3, "Potential Loyalists",

&#x20;   CustomerRFM\[R\_Score] <= 2 \&\& CustomerRFM\[F\_Score] >= 3, "At Risk",

&#x20;   CustomerRFM\[R\_Score] <= 2 \&\& CustomerRFM\[F\_Score] >= 4, "Can't Lose Them",

&#x20;   CustomerRFM\[R\_Score] = 1  \&\& CustomerRFM\[F\_Score] = 1,  "Lost",

&#x20;   "Others"

)

\## نحوه اجرا



۱. مخزن را کلون کنید:

&#x20;  ```bash

&#x20;  git clone https://github.com/your-username/digikala-analytics-dashboard.git

&#x20;  ```

۲. فایل‌های CSV را در پوشه `/data` قرار دهید

۳. فایل `Digikala\_Dashboard.pbix` را در Power BI Desktop باز کنید

۴. در Power Query مسیر فایل‌ها را به پوشه محلی `/data` خود تغییر دهید

۵. روی \*\*Refresh\*\* کلیک کنید — تمام جداول و معیارها به‌طور خودکار پر می‌شوند
